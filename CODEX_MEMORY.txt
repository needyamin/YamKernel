Positioning
-----------
YamOS is an exceptional (not “experimental toy”) x86_64 OS: YamKernel graph hybrid,
Limine boot, YamGraph resource graph, kernel-first desktop/services. Marketing tone:
ambitious engineering, honest gaps—see README “Not Complete Yet”.

Repo / build
------------
Root: C:\laragon\www\YamOS  |  WSL: /mnt/c/laragon/www/YamOS
Build: wsl -d Ubuntu -- bash -lc "cd /mnt/c/laragon/www/YamOS && make iso"
Verify: powershell -ExecutionPolicy Bypass -File .\scripts\test.ps1 verify
Artifacts: build/yamkernel.iso  |  build/verify.log

Rules
-----
- Kernel/OS foundations before app theater: no fake runtimes, stub installers, or
  userland papering over missing kernel ABI.
- If a feature needs kernel support, implement kernel first.
- Do not run raw qemu-system-x86_64; use test.ps1 helpers.

Architecture (primary paths)
----------------------------
main/shell/init: src/kernel/main.c, src/kernel/shell.c, src/os/bin/init.c
ELF/VFS/syscalls: src/fs/elf.c, src/fs/vfs.c, src/cpu/syscall.c,
  ABI header src/kernel/api/syscall.h (SYS_MAX=94; syscall numbers + OS flags)
Poll: src/fs/poll.h, sys_poll in vfs.c
Drivers: src/drivers/ (pci.c, driver_manager, block, e1000, xhci, ahci, virtio-blk)
Net: src/net/  |  Mem: src/mem/  |  Compositor: src/os/services/compositor/
Apps: src/os/services/app_registry/  |  libyam: src/os/lib/libyam/  |  libc: src/os/lib/libc/

Current capability (summary — detail drifts; verify in tree)
-------------------------------------------------------------
- QEMU path: PCI inventory, FAT32 block-backed mounts under /mnt, VFS promotion of
  persistent dirs when vd0 present, e1000 + DHCP/DNS/TCP/HTTP verified in verify.log.
- TLS: store bootstrap + ClientHello probe; full HTTPS handshake not done.
- Threads/signals/futex/pthread pieces and SYS_SELECT/SYS_FCNTL nonblocking I/O:
  see syscall.h + Phase notes in README.
- Desktop: compositor-native Wayland-style UI, multi-user, File Manager, browser
  (plain HTTP path), settings windows—see README “Current Desktop Behavior”.
- Major gaps: execve/dynamic linker, full TLS, portable Linux userspace, browser-
  class stack—README lists foundations still required.

Roadmap (compressed)
--------------------
Phases 1–2 complete; Phase 3 in progress. Open work includes full TLS/HTTPS,
ypkg, sigframe/ucontext, ELF dynamic linker, execve/fork/Unix sockets,
virtio-gpu/Mesa path, libc port, eventual Chromium-class browser—see README table.
