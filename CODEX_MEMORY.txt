Owner Direction
---------------
- Kernel/OS-level work first. No fake runtimes, package-manager theater, or
  language-specific stubs. App-level work is unblocked only after kernel
  foundations are solid (see missing compat list below).
- Kernel-first rule: if a feature needs missing kernel support, fix the kernel
  first, then build the feature on top.
- YamOS is not complete enough to run arbitrary x86_64 software yet.
- Standing required foundations (incomplete):
  - execve + strong process model
  - dynamic linker / PT_INTERP / dlopen
  - pthread ABI, TLS, thread-local errno, robust futex
  - signal delivery, masks, handlers, interruption semantics
  - persistent writable filesystem (FAT32 block-backed fully mounted)
  - full POSIX path/fcntl/lock/permission semantics
  - nonblocking sockets (DONE: SOCK_NONBLOCK, select, fcntl O_NONBLOCK)
  - select/poll/epoll (DONE: select via sys_poll, SYS_SELECT=92, SYS_FCNTL=93)
  - TLS/HTTPS + certificate store (partial: probe only)
  - package manager/downloader
  - PTY, termios, job control
  - UID/GID, file modes, capabilities, sandboxing
  - GPU, USB audio, Wi-Fi 802.11 MAC, Bluetooth HCI transport
  - browser engine: DOM, CSS, JS VM, TLS, codecs, sandboxing

- Built-in drivers: e1000 8086:100E, xhci 0C:03, vga 03:00, ahci 01:06,
  virtio-blk 1AF4:1001, virtio-blk-modern 1AF4:1042, iwlwifi (class 02:80).
- Block core: initialized, max_devices=16. FAT32 volumes auto-mounted at /mnt.
- VFS promotes /home, /var, /usr/local to persistent FAT32 when vd0 present.
- e1000: MMIO mapped, TX/RX rings ready, link up.
- DHCP: ACK IP=10.0.2.15 GW=10.0.2.2 DNS=10.0.2.3 (verified).
- HTTP: GET http://ansnew.com/ succeeds (797-797 bytes, varies by DNS).
- TLS: certificate-store bootstrap + bounded ClientHello probe only; full
  TLS handshake not yet complete.
- Wi-Fi: iwlwifi driver skeleton; reports no adapter/firmware in QEMU.
- Bluetooth: hci_usb skeleton; reports no USB HCI transport in QEMU.

Current Filesystem State
------------------------
- VFS mounts: /tmp (ramfs), /var (ramfs→FAT32), /usr/local (ramfs→FAT32),
  /opt, /home (ramfs; per-user subdirs created at login), /mnt (block FAT32).
- Supported: mkdir, open (O_CREAT, O_EXCL, O_TRUNC, O_APPEND), read, write,
  unlink, readdir, rename (RAMFS in-place; FAT32 copy+unlink), stat/fstat,
  ftruncate, lseek (SEEK_END via fstat), chdir/getcwd, *at dirfd family.
- O_EXCL: rejects if path exists (lock-file support).
- O_APPEND: moves offset to EOF before write.
- VFS open checks existence before fd alloc; O_CREAT required for new files.
- 1MB LRU block cache with dirty tracking.
- /proc pseudo-files: cpuinfo, meminfo, version, uptime, loadavg.
- /dev: devfs_exists() guards open.

Current Network Stack State
----------------------------
- TCP state machine: SYN, SYN-ACK, ACK, CLOSE_WAIT, FIN handling.
- tcp_sock_t.nonblocking flag; tcp_set_nonblock() / tcp_is_nonblock().
- connect(): returns -EINPROGRESS if nonblocking.
- recv(): returns -EAGAIN if nonblocking and no data.
- accept(): returns -EAGAIN if nonblocking and no client.
- sys_socket_user(): handles SOCK_NONBLOCK flag (ORed with type).
- sys_fcntl_user(): F_GETFL / F_SETFL O_NONBLOCK on socket fds.
- sys_select_user(): backed by sys_poll; 128-bit fd_set (2×u64);
  maps readfds/writefds to POLLIN/POLLOUT; writes back result sets.
- libc sys/socket.h: fd_set, FD_ZERO/SET/CLR/ISSET, select(), fcntl(),
  SOCK_NONBLOCK, EAGAIN, O_NONBLOCK, EINPROGRESS.
- TCP socket file_ops: read/write/poll/close wired through VFS file_t.
- UDP datagram fd-backed ABI: reserved (SYS_SENDTO/RECVFROM stub -1).

Current Threading & Signal State  (Phase 1.1 complete)
-------------------------------------------------------
- sched_thread_create(): copies parent task, shares pml4, allocates new kernel
  stack; IA32_FS_BASE written from task.tls_base at every context switch.
- TLS block (tls_block_t): allocated by pthread_create, FS:0 = self pointer
  (glibc ABI compat), 128 pthread_key slots in specific[].
- Futex: FUTEX_WAIT/WAKE hash table (64 buckets, wait.c); SYS_FUTEX=42.
  FUTEX_WAIT/WAKE/WAKE_PRIVATE/WAIT_PRIVATE constants in syscall.h + libyam.
- pthread_mutex_t: futex-backed 0/1/2 (unlocked/locked/contended);
  lock uses CAS(__ATOMIC_ACQUIRE) + futex_wait; unlock = atomic_fetch_sub + wake.
- pthread_cond_t: sequence-counter futex; cond_wait = unlock + futex_wait on seq
  + relock; broadcast = fetch_add + futex_wake(INT_MAX).
- pthread_key_create/delete/getspecific/setspecific: backed by tls_block.specific[].
- Signal ABI: SYS_SIGACTION=89, SYS_SIGPROCMASK=90, SYS_SIGRETURN=91.
  sigaction stores handler; SIGKILL(9)/SIGSTOP(17) cannot be caught/blocked.
  sigprocmask: SIG_BLOCK/UNBLOCK/SETMASK; delivers at every SYSCALL_RETURN.
  Default unhandled signal action: sched_exit_current(128+sig).
  sys_sigreturn: first slice only (full ucontext push pending).
- New OS flags: YAM_OS_FLAG_PTHREAD_ABI (1<<10), YAM_OS_FLAG_SIGNALS (1<<11).

Current Application Architecture State
---------------------------------------
- App registry: 64 slots, spinlock-protected. Records PID, task name,
  YamGraph node, app type, permissions, name, publisher, version, desc.
- Native SDK: src/os/lib/libyam/app.h — yam_os_info(), yam_app_register(),
  yam_app_query(), yam_spawn(), yam_spawn_args(), yam_spawn_env().
- libc spawn wrappers: spawn(), spawnv(), spawnve(); execve() = ENOSYS.
- ELF spawn: elf_spawn_path_argv_envp(); bare-name resolver: /bin,
  /usr/local/bin, /opt/yamos/packages, /home/root/bin.
- Exit/wait: TASK_ZOMBIE → sched_waitpid() reap; SYS_WAITPID copies
  encoded status to user via copy_to_user(); WNOHANG supported.
- /bin/hello: boot-probe ELF; exercises stat, fstat, ftruncate, rename,
  O_APPEND, O_EXCL, *at dirfd calls, spawn+waitpid from Ring 3.
- authd: registers as YamOS service (type=2, perm=IPC) before YamGraph IPC.
- FAT32 install: PID 1 writes /usr/local/bin/hello-local persistently.

Current Desktop/GUI State
--------------------------
- Compositor: Wayland-style kernel-space, 1280x800, DRM scanout.
- States: SETUP → LOGIN → DESKTOP → VTTY.
- Multi-user: YAM_MAX_USERS=8; yam_user_account_t (username, password hash,
  display_name, admin, active); current_user[32] tracks logged-in user.
- Login: validates via wl_password_hash; yam_channel_t authd fallback.
- Setup: creates accounts for setup_user, root, guest; persists via FAT32.
- Home dir: /home/<username> created by sys_mkdir at setup/login time for
  each account; skip-setup creates /home/root + /home/guest.
- File Manager (wl_file_manager.c):
  - Opens in /home/<current_user> (resolved from comp->current_user at launch).
  - Sidebar: Home (user-specific), Users (/home), Root (/), Temp, System, Volumes.
  - Browse, search, sort (name/type/size), breadcrumb address bar, history nav.
  - New file (O_CREAT|O_TRUNC), new folder (mkdir), delete (unlink), open/edit,
    save text, keyboard navigation (Up/Down/PgUp/PgDn/Enter/Backspace).
  - Text editor mode: read, edit, save back to VFS.
- Browser (wl_browser.c): real HTTP via http_get(); typed render-tree (headings,
  paragraphs, lists, links, pre/code); scrollable viewport; HTTPS rejected until
  TLS ready; redirect follow (plain HTTP, up to 2 hops).
- Settings windows (standalone, open-or-focus): Ethernet, Wi-Fi, Bluetooth,
  Sound, Display. Top-bar chips open respective window directly.
- DRM: drm_page_flip_damage() (dirty-rect updates), drm_set_mode() (Bochs VBE
  I/O port modesetting; live 1024x768 ↔ 1920x1080 toggle).
- Launcher: two-column grid (apps + settings); single-instance focus helper.
- Window chrome: 36px titlebar, minimize/maximize/restore/close, drop-shadow,
  maximize scales content + remaps input coords.
- IPC: YamGraph channels; authd bidirectional auth flow.
