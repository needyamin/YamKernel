1. Project Vision & "Special" Features

YamOS is an Adaptive Microkernel operating system. Unlike traditional designs, it is built on the philosophy that "Hardware is just a data node."

What makes YamOS special?
  • YamGraph System: Instead of simple files or permissions, YamOS uses a directed graph to represent resources. To access a NIC or a screen buffer, a process must be "linked" to that node in the graph.
  • Hot-Swap Drivers: Because drivers run in Ring 3 as standard ELFs, you can crash, upgrade, or restart a Video or Network driver without ever rebooting the kernel.
  • Phase-Boot Architecture: A deterministic 7-phase boot process ensures system integrity via continuous self-tests.

4. Kernel Architecture (Ring 0)

Memory Layout

  • Kernel Base: 0xffffffff80000000 (Linked at -2GB)
  • Physical Direct Map: 0xffff800000000000 (Access all RAM)
  • Allocation: Uses a Cell Allocator for physical pages and a Slab/Cache system for kernel structures (like task_t).

Scheduling & Multiprocessing

The kernel wakes all CPUs found by Limine. Each CPU runs its own instance of the CFS-lite scheduler. Load balancing is handled via task migration during sched_tick.

5. OS & Services (Ring 3)

The Driver Isolation Model

Under the "Modern Tech" vision, drivers are not linked to the kernel. They are compiled as standalone ELFs and loaded by the kernel into Ring 3 address spaces. They interact with hardware via specific "Privileged Syscalls" (e.g., SYS_PCI_CONFIG_READ).

Wayland Compositor

The compositor (src/os/services/compositor) is the owner of the physical framebuffer. It provides a shared memory interface to apps for high-speed rendering without kernel intervention.

6. How to Recode & Develop

Follow these steps to contribute or modify the system:

A. Setting Up the Environment

You need gcc, ld, nasm, make, and xorriso. We recommend a WSL (Ubuntu) or Linux environment.

B. Adding a New Ring 3 App

  1. Create src/os/apps/my_app.c.
  2. Include "yam.h".
  3. Add your app to the Makefile in the APPS list.
  4. Call make apps and make iso.

C. Debugging

Use the serial log. YamOS mirrors all kernel and user console output to the serial port (COM1). You can view it in your terminal when running with make run.
