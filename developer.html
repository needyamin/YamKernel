2. High-Level Architecture

YamKernel is a monolithic-leaning hybrid kernel: it keeps memory, scheduling, IRQs, and core IPC in the kernel for performance, while exposing user-mode tasks via a fast SYSCALL path. A graph (YamGraph) sits underneath, modeling every resource (task, memory cell, device, channel, capability) as a node with typed edges.

+----------------------------- Ring 3 (user) ------------------------------+
|  user-demo program / future apps  |  POSIX-ish runtime (planned)         |
+----------------------------- SYSCALL / SYSRET ---------------------------+
|  syscall_dispatch  (sys_write / exit / getpid / yield / sleepms)         |
+--------------------------- Ring 0 services ------------------------------+
|  Shell (task #0)  Scheduler (CFS-lite)  Wait queues / mutex / sleep      |
+--------------------------- Core kernel core -----------------------------+
|  YamGraph  | IPC (channels, capabilities) | VFS / NET / IPC scaffolds    |
+--------------------------- Memory + CPU ---------------------------------+
|  PMM (Cell alloc) | VMM (4-level paging) | Heap | Slab | Per-CPU data    |
|  GDT/IDT/TSS | APIC + APIC timer | ACPI/MADT | SYSCALL MSRs | Security   |
+---------------------------- Hardware (Limine boot) ----------------------+
|  CPU(s) | RAM | LAPIC/IOAPIC | PCI/USB/I2C/SPI | NIC | Display/FB | KBD  |
+--------------------------------------------------------------------------+

3. Pros & Cons

Pros

  • Modern x86_64 base, Limine v3, NX/SMEP/SMAP/UMIP/WP enabled by default.
  • Fast SYSCALL/SYSRET path (no int 0x80 overhead).
  • Per-CPU data via GS_BASE — Linux-grade ergonomics.
  • Slab + cell + heap allocators give tunable allocation classes.
  • CFS-lite + sleep/wake/mutex primitives in-tree.
  • YamGraph offers a cleaner permission model than UID/GID.
  • SMAP-aware syscall handlers (auto STAC/CLAC).
  • Hybrid model: monolithic speed + microkernel-style IPC primitives.
  • Boot/dev cycle is fast (single Makefile; auto-discovers .c/.asm).
  • YamBoot menu (Normal / Safe Mode / Reboot) for triage boots.

Cons

  • SMP AP execution not yet wired (only BSP runs code).
  • No ELF loader for user apps yet (raw blob copy demo only).
  • VFS/Net/IPC are scaffolds — no real disk/FS or live TCP yet.
  • No demand paging, no copy-on-write, no fork/exec.
  • No POSIX/C library; user-side ABI minimal.
  • Driver model is in-kernel only; no Ring-3 driver isolation yet.
  • Limited hardware coverage (PS/2 KBD, e1000 stub, framebuffer).
  • Scheduler runs single-CPU only until SMP lands.
  • No formal security audit / no signed boot chain.

4. Kernel Architecture (Ring 0)

4.1 Kernel Layout

  • Higher-half kernel at 0xffffffff80000000 (linker.ld).
  • HHDM at 0xffff800000000000 from Limine.
  • Per-CPU structure addressed via GS_BASE (kernel) / KERNEL_GS_BASE (user).
  • MMIO virtual window at 0xFFFFD00000000000 (with fallback windows) used by APIC/IOAPIC mapping.

4.2 CPU Init Path (in order)

  1. gdt_init — 5 segments (null, kernel CS/SS, user CS/SS) + TSS for ring transitions.
  2. idt_init — 32 exception vectors + 16 IRQ vectors (48 total).
  3. cpuid_init — vendor/family/model/feature scan.
  4. security_init — enables NX (EFER.NXE), WP (CR0.WP), SMEP/SMAP/UMIP/FSGSBASE (CR4).
  5. vmm_init + pmm_init + heap_init — memory online.
  6. acpi_init → apic_init → ioapic_init → percpu_init → smp_init.
  7. syscall_init — STAR / LSTAR / SFMASK / EFER.SCE configured.
  8. yamgraph_init — kernel subsystems registered as graph nodes.
  9. sched_init + APIC timer + shell_start.

4.3 Privilege Model

  • Ring 0: kernel.
  • Ring 3: user tasks (loaded as raw blobs today; ELF loader planned).
  • Transition: SYSCALL in (saves user RSP, swaps GS), SYSRET out.
  • IRQ-from-Ring 3: isr_common auto-swapgs on entry/exit when CS RPL=3.
  • SMAP: sys_write uses STAC/CLAC around user buffer reads.

5. OS Architecture (Ring 3 / Userland)

YamOS-on-YamKernel is intentionally minimal today — the goal is correctness of the kernel/user boundary first, then layering services on top.

5.1 Userland Today

  • Position-independent demo program (src/sched/user_demo.asm) running at virtual 0x400000.
  • User stack at 0x800000, single 4 KiB page.
  • Issues SYS_WRITE + SYS_SLEEPMS(250) in a loop.
  • Toggle via #define YAM_DEMO_TASKS 1 in kernel/main.c.

5.2 Userland Targets (planned)

  • ELF64 static loader (fs/ + sched/user.c hooks).
  • Minimal libc (string, IO via syscalls).
  • Wayland-style compositor that owns the framebuffer; apps get shared-memory surfaces.
  • Ring-3 driver framework backed by capability tokens.

6. Memory Subsystem

6.1 PMM — Cell Allocator

Physical pages are tracked by a fractal quad-tree:

  • Root cell = entire physical RAM.
  • Split: each cell divides into 4 children when partially allocated.
  • Merge: when all 4 siblings are free, they coalesce.
  • Each cell can carry an owner reference (a YamGraph node id).

6.2 VMM

  • 4-level paging (PML4 → PDPT → PD → PT).
  • Walker is huge-page aware: refuses to descend into existing 2 MB / 1 GB entries (e.g. Limine HHDM).
  • vmm_map_page issues invlpg after writing the leaf.
  • MMIO uses a dedicated upper window so it never collides with HHDM huge pages.

6.3 Heap

Bump-style heap initialized at 0xffff800000XXXXXX (HHDM-mapped). Used for early kernel allocations and as a backing store for slab metadata.

6.4 Slab Allocator

  • kmem_cache_create(name, obj_size, align)
  • kmem_cache_alloc(cache) — O(1) freelist pop.
  • kmem_cache_free(cache, obj) — O(1) push.
  • Each slab = 1 PMM page; objects threaded via their first 8 bytes.
  • Currently used for task_t allocations.

7. Scheduler & Concurrency

7.1 CFS-lite

  • Single ready list; pick task with the lowest vruntime.
  • vruntime += 1024 / weight per timer tick.
  • Weights by priority: 8 / 4 / 2 / 1.
  • New tasks inherit the minimum vruntime to avoid monopolizing.
  • Per-task kernel stack: switch_to() updates TSS.rsp0 and per-CPU kernel_rsp.

7.2 Sleep / Wake

  • task_sleep_ms(ms) — block on the sleeping list.
  • sched_tick wakes any sleepers whose wakeup_tick <= now.
  • Wait queue: wq_sleep / wq_wake_one / wq_wake_all.
  • mutex_t = atomic xchg fast path + wait queue slow path.
  • IRQ-safe spinlock with pushfq/cli/popfq pattern.

7.3 Preemption

APIC timer at 100 Hz drives sched_tick → sched_yield. In stability mode (YAM_PREEMPTIVE = 0) preemption is disabled and the shell runs cooperatively. Flip the macro in kernel/main.c to enable it.

8. Interrupts & APIC

  • Legacy 8259 PIC fully masked once APIC is online.
  • LAPIC + IOAPIC MMIO mapped through verified mapper (tries multiple virtual windows, validates vmm_virt_to_phys before use, falls back to HHDM if already valid, returns NULL on total failure).
  • APIC timer calibrated via PIT channel 2 polling — IRQ-free.
  • EOI policy: APIC mode auto-EOIs all vectors except 0x20 (timer self-EOIs in handler).
  • From Ring 3, ISR stub isr_common performs swapgs on entry/exit (RPL check on saved CS).

9. Syscall ABI

SYSCALL convention: rax=number, args in rdi, rsi, rdx, r10, r8, r9. Return in rax (signed).

NRNameArgsBehavior
1SYS_WRITEfd, ubuf, lenWrites to serial; SMAP-aware.
2SYS_EXITcodeMarks task TASK_DEAD, yields, halts.
3SYS_GETPID—Returns current task id.
4SYS_YIELD—Voluntary reschedule.
5SYS_SLEEPMSmsBlock current task on sleeping list.

Defined in src/cpu/syscall.h; max ID reserved = 16.

10. YamGraph Resource Model

  • Nodes: tasks, memory regions, devices, files, channels, IRQs, namespaces.
  • Edges: OWNS, CAPABILITY, CHANNEL, MAPS, DEPENDS.
  • Capabilities: unforgeable tokens carried by edges.
  • Channels: typed bidirectional graph edges for IPC.
  • The shell exposes the graph via graph command.

Compared to UID/GID, this lets you revoke access cascade-style by removing edges and lets the scheduler/IPC reason about resource topology directly.

11. Driver Model

  • Today: in-kernel drivers (Ring 0). Files under src/drivers/.
  • Working: serial COM1, framebuffer, PIT, RTC, PS/2 keyboard.
  • Scaffold: PCI enumeration, USB stack, I2C, SPI, e1000 NIC, iwlwifi, Bluetooth HCI, mouse.
  • Future: Ring-3 driver runtime — drivers loaded as ELFs, talk to hardware via privileged syscalls (SYS_PCI_*, SYS_MAP_MMIO) gated by capabilities.

12. Boot Pipeline

Limine (BIOS or UEFI) -> kernel_main()
  Phase 1  serial_init  -> framebuffer init
  Phase 2  YamBoot menu (Normal / Safe / Reboot)
  Phase 3  GDT / IDT / CPUID / security_init
  Phase 4  VMM / PMM / Heap
  Phase 5  ACPI -> LAPIC -> IOAPIC -> percpu -> smp -> syscall_init
  Phase 6  YamGraph init + register kernel subsystems
  Phase 7  Self-tests (PMM + YamGraph)
  Phase 8  Drivers (PCI/USB/I2C/SPI/VFS/IPC/NET/MOUSE) - skipped in Safe Mode
  Phase 9  Scheduler init  (preemption optional)
  -> shell_start()  // runs as task #0

15. Debug Playbook

15.1 Panic Triage

VectorHint
#13 GP faultBad segment selector / non-canonical address / GDT misalignment for SYSRET.
#14 Page faultRead CR2 (faulting VA). Bit 0 of error: 0=not present, 1=protection. Bit 1: 0=read, 1=write. Bit 2: 0=kernel, 1=user.
#6 Invalid opcodeLikely jumped into garbage stack. Inspect RIP + nearby code.
#8 Double faultProbably bad TSS.rsp0 or recursion in early IDT path.

15.2 Common Causes by CR2 range

  • 0x000000… (low): NULL deref in kernel.
  • 0xffff800000000xxx: HHDM region — likely raw access to phys not yet mapped.
  • 0xffff8000fexxxxxx: APIC/IOAPIC MMIO — verify the dedicated MMIO window mapped successfully.
  • 0x400000–0x800000: user-space demo region — check page flags or SMAP.

15.3 Useful Steps

  • Always read serial output (make run mirrors to stdio).
  • Disable YAM_DEMO_TASKS + YAM_PREEMPTIVE first when bisecting boot regressions.
  • For map-failure suspects: log the return value of vmm_map_page and re-verify with vmm_virt_to_phys.
  • Reduce visual noise by booting Safe Mode — it skips PCI/USB/I2C/SPI/VFS/IPC/NET/MOUSE.

17. Roadmap

Tier 1 — Stabilize

  • SMP AP startup (INIT-SIPI-SIPI).
  • ELF64 user loader.
  • Per-CPU run queues.
  • Better panic dumps (stack trace, symbol table).

Tier 2 — Beat-Linux Targets

  • Demand paging + CoW.
  • Real VFS (FAT32 + ext2).
  • TCP/IP fast path.
  • Capability-based fork/exec.
  • Compositor in Ring 3 with shared-memory surfaces.

Tier 3 — Frontier

  • RCU.
  • eBPF-style in-kernel programs.
  • Formal verification on critical paths.
  • NUMA-aware slab.
  • Signed boot chain + measured boot.
