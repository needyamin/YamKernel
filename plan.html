Critical Technical Gaps

HIGH Threads & Signals

Status: MISSING

Essential for modern language runtimes and multi-process browsers. Without this, application complexity is capped at single-threaded logic.

HIGH Dynamic Linking

Status: MISSING

Currently, every app bundles its own libc. An ELF interpreter is required to enable shared libraries and reduce system-wide memory pressure.

Non-blocking I/O

Status: MISSING

The current networking stack is synchronous. Implementing select() or epoll() is vital for responsive network applications.

Page Cache

Status: MISSING

A massive bottleneck for disk-intensive apps. Implementing an LRU page cache will make the system feel significantly "snappier" on real hardware.

Implementation Roadmap

Phase 1: The Runtime Foundation

Focus on kernel stability and porting ability.

  • Implement Kernel Threads & Futexes
  • Build ELF Dynamic Loader (ld.so)
  • Standard POSIX Signal Delivery

Phase 2: Performance & Hardware

Moving from a VM-only demo to a real-world OS.

  • LRU Page Cache for Storage
  • AHCI / NVMe Driver Hardening
  • VirtIO-GPU / DRM Modesetting improvements

Phase 3: Connected Ecosystem

Enabling web connectivity and package management.

  • Non-blocking Socket API
  • Full TLS/HTTPS Handshake Completion
  • Initial Package Manager (ypkg)

Phase 4: Experience Polish

Finalizing the macOS-inspired desktop experience.

  • FreeType / HarfBuzz Font Integration
  • Global Theme Service (CSS-based)
  • VFS-level Permissions (UID/GID enforcement)
