WHAT IS YAMKERNEL?
------------------
YamKernel is your own operating system kernel — the core brain of an OS.
It is 100% original code. No Linux code. No Windows code. No copied code
from any existing OS on the planet. Every single line is written from scratch.

YamKernel introduces a completely new idea: the YamGraph Resource Graph.
In this system, everything (processes, memory, devices, files) is a "node"
in a live graph, and permissions flow through "edges" as capability tokens.
No other operating system in the world uses this architecture.


WHAT HAPPENS WHEN YOU BOOT YAMKERNEL?
-------------------------------------
When you boot YamKernel (from ISO in VMware, VirtualBox, or bare metal):

  1. The Limine bootloader loads YamKernel into memory
  2. The screen turns dark (dark background: #0A0A14)
  3. You see the YamKernel ASCII art logo in cyan/yellow colors
  4. Boot messages scroll showing each subsystem initializing:

     === Phase 1: CPU Setup ===
     [GDT] Loaded: 5 segments + TSS

     === Phase 2: Memory (Cell Allocator) ===

     [PMM] Cell Allocator initializing...
     [PMM] Total usable: XXX MB
     [HEAP] Initialized: 256 KB

     === Phase 3: YamGraph Resource Graph ===
     [YAMGRAPH] Initialized — root kernel node created
     [YAMGRAPH] Kernel subsystems registered

     === Phase 4: Self-Tests ===
     [PMM] Cell Allocator self-test: PASS
     [YAMGRAPH] Resource Graph self-test: PASS

     ╔══════════════════════════════════════╗
     ║   YamKernel v0.2.0 — BOOT COMPLETE  ║
     ╚══════════════════════════════════════╝

  5. The kernel drops you into the INTERACTIVE DEBUG TERMINAL (`yam>`)


DOES IT HAVE A GUI?
-------------------
Current version (v0.2.0): NO graphical desktop yet, but it HAS A TERMINAL!

YamKernel v0.2.0 boots to a highly interactive TEXT CONSOLE with colored text.
It runs a "Kernel REPL" (interactive shell) where you can manage the core engine.

Commands you can run right now:
  - `top`      : Displays a beautiful ASCII dashboard of Memory and YamGraph stats.
  - `graph`    : Dumps a visual tree of the entire YamGraph OS architecture.
  - `mem`      : Shows Cell Allocator memory blocks.
  - `shutdown` : Safely powers off VirtualBox/VMware/QEMU.
  - `reboot`   : Restarts the CPU.

Future versions will add:
  - v0.3.0: Filesystem (NexusFS concept)
  - v0.4.0: Userspace programs (loading custom .elf files)
  - v1.0.0: Native GUI desktop environment mapping the Graph visually.

HOW IS YAMKERNEL DIFFERENT FROM EVERY OTHER OS?
-----------------------------------------------
1. YAMGRAPH (No other OS has this)
   - Every resource is a node in a live directed graph
   - Permissions are edges, not file permissions
   - The scheduler will use graph topology for decisions
   - IPC channels are visible graph edges you can inspect

2. CELL ALLOCATOR (Our own memory algorithm)
   - Memory is managed as a fractal quad-tree
   - Each "cell" can split into 4 children
   - Each cell knows which graph node owns it
   - This is NOT the buddy system, NOT slab allocation

3. CAPABILITY TOKENS (Not UID/GID like other OSes)
   - Permissions are unforgeable tokens
   - You can delegate tokens to other processes
   - Revoking a token cascades through the entire graph
   - No root user — everything is capability-based

4. CHANNELS (Not pipes or sockets)
   - IPC through typed bidirectional channels
   - Channels are graph edges — visible and revocable
   - Ring-buffered for high performance
   - Permission-checked via YamGraph automatically


HOW TO INSTALL / BOOT
---------------------
Step 1: Build the ISO (requires WSL/Ubuntu on Windows)
   $ cd /mnt/c/laragon/www/kernel
   $ make setup      (install build tools — one time only)
   $ make iso        (creates build/yamkernel.iso)

Step 2: Boot the ISO
   Option A - QEMU:      $ make run
   Option B - VMware:    Create VM > attach yamkernel.iso as CD > Boot
   Option C - VirtualBox: Create VM (64-bit Other) > attach ISO > Boot
   Option D - USB:       $ sudo dd if=build/yamkernel.iso of=/dev/sdX


TECHNICAL SPECS
---------------
  Architecture:     x86_64 (64-bit)
  Language:         C11 + x86_64 Assembly (NASM)
  Bootloader:       Limine (BSD licensed, NOT part of YamKernel)
  Memory model:     Higher-half kernel at 0xFFFFFFFF80000000
  Page size:        4 KB
  Max nodes:        4,096 graph nodes
  Max edges:        8,192 graph edges
  Max channels:     256 IPC channels
  Initial heap:     256 KB (expandable)
  Serial output:    COM1 @ 115200 baud

  External code:    NONE (only limine.h boot protocol header)
  Code from Linux:  NONE
  Code from Windows: NONE
  Code copied:      NONE — 100% original
